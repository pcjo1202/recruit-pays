# 프로젝트 구조
> Feature-Sliced Design(FSD) 기반으로 구현

## 설계 기준
- Next.js App Router, Server Components 최우선 사용
- 유저 인터렉션이 있는 컴포넌트는 Client Components로 구현
  
## 레이어 책임 정의
| Layer | 경로 | 책임 |
| --- | --- | --- |
| app | `app/` | 라우트 매핑, 레이아웃, 메타데이터, 서버 컴포넌트에서 데이터 요청 |
| widgets | `widgets/` | 페이지 단위 UI 조합, 여러 feature/entities를 병합 |
| features | `features/` | 사용자 행동 단위 UI·비즈니스 로직 |
| entities | `entities/` | 도메인 모델, API 모듈, DTO→Domain 변환, 서버 액션 |
| shared | `shared/` | 공통 UI, lib, config, types, 스타일 토큰 |

## 컴포넌트 분류 기준
- **Client Component 조건**
  - 사용자 입력
  - 차트 상호작용
- **폴더 규칙**
  - `widgets/<page>/<section>`
  - `features/<domain>/<feature>`
  - `entities/<domain>/{api,model,lib}`
  - `shared/ui/<component>`
